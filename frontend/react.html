    # Class 中的 State






     





     







     





    import React from 'react';

class Student extends React.Component {
  constructor(props) {
    super(props);
    this.state = {count: 1};
    // 确保函数可以访问组件属性（ES2015）
    this.click = this.click.bind(this);
  }
  click() {
    const count = this.state.count;
    this.setState({ count: count + 1})
  }
  render() {
    return (
      <div>
        <button onClick={this.click}>
          点击我
        </button>
        <p>您点击了{this.state.count}次</p>
      </div>
    );
  }
}

    # Profiler

    测量一个 React 应用多久渲染一次以及渲染一次的 代价

    <Profiler id="Navigation" onRender={callback}>
  <Navigation {...props} />
</Profiler>
    1
    2
    3

    为了分析 Navigation 组件和它的子代。应该在需要时才去使用它。

    :- :-
    id(string) 发生提交的 Profiler 树的 id
    onRender(function) 组件树任何组件 “提交” 一个更新的时候调用这个函数

    # onRender 回调函数

    :- :-
    phase: "mount" \| "update" 判断是由 props/state/hooks 改变 或 “第一次装载” 引起的重渲染
    actualDuration: number 本次更新在渲染 Profiler 和它的子代上花费的时间
    baseDuration: number 在 Profiler 树中最近一次每一个组件 render 的持续时间
    startTime: number 本次更新中 React 开始渲染的时间戳
    commitTime: number 本次更新中 React commit 阶段结束的时间戳
    interactions: Set 当更新被制定时，“interactions (opens new window)” 的集合会被追踪

    # 生命周期

    # 挂载

    方法 描述
    constructor (props) 渲染前 # (opens new window)
    static getDerivedStateFromProps() 调用 render 方法之前调用 # (opens new window)
    render() class 组件中唯一必须实现的方法 # (opens new window)
    componentDidMount() 在组件挂载后(插入 DOM 树中)立即调用 # (opens new window)
    UNSAFE_componentWillMount() 在挂载之前被调用，建议使用 constructor() # (opens new window)

    在 constructor() 上设置初始状态。在 componentDidMount() 上添加 DOM 事件处理程序、计时器（等），然后在 componentWillUnmount() 上删除它们。

    # 卸载

    方法 描述
    componentWillUnmount() 在组件卸载及销毁之前直接调用 # (opens new window)

    # 过时 API

    过时方法 新方法
    componentWillMount() UNSAFE_componentWillMount() # (opens new window)
    componentWillReceiveProps() UNSAFE_componentWillReceiveProps() # (opens new window)
    componentWillUpdate() UNSAFE_componentWillUpdate() # (opens new window)

    17+ 之后不再支持，在 17 版本之后，只有新的 UNSAFE_ 生命周期名称可以使用。

    # 更新

    方法 描述
    static getDerivedStateFromProps(props, state) 调用 render 之前调用，在初始挂载及后续更新时都会被调用 # (opens new window)
    shouldComponentUpdate(nextProps, nextState) 如果返回 false，则跳过 render() # (opens new window)
    render() 在不修改组件 state 的情况下，每次调用时都返回相同的结果 # (opens new window)
    getSnapshotBeforeUpdate() 在发生更改之前从 DOM 中捕获一些信息（例如，滚动位置） # (opens new window)
    componentDidUpdate() 这里使用 setState()，但记得比较 props。首次渲染不会执行此方法 # (opens new window)

    # 错误处理

    方法 描述
    static getDerivedStateFromError(error) 后代组件抛出错误后被调用，它将抛出的错误作为参数，并返回一个值以更新 state # (opens new window)
    componentDidCatch(error, info) 在后代组件抛出错误后被调用，会在“提交”阶段被调用，因此允许执行副作用 # (opens new window)
